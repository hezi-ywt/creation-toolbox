# 成品形态参考

故事开发到一定阶段，创作者可能想"整理一下"。这份文件不强制任何模板——给几种典型形态做参考，让你在跟创作者讨论时心里有数。

**核心原则**：
- 这些是**素材**不是**模板**——AI 不要按固定结构填空
- 用途决定形态——给制片人看 vs 给自己回顾，需要的不一样
- 永远问用途，再选形态——不要默认"成品 = Logline + Premise + Synopsis"

## 何时不要整理

- 故事还在挖矿期（还没稳定的 Logline）—— 强行总结会污染创作者的思路
- 创作者还在大改 —— 等他自己说"先这样了"
- 创作者只是想看草稿状态 —— 直接指他看 草稿 段，不要包装

## 几种典型用途

### A. 给自己 / 创作团队回顾（最常见）

**特点**：快速对齐，不必完整

**包含什么**（按需）：
- 当前最新 Logline
- 关键已定事实（直接 quote 已定段几条）
- 当前未决的关键问题

**例子**：
```
# 雨夜便利店 · 当前状态（2026-05-17）

Logline: 一个失去儿子的老人每周三在便利店遇到陌生人 Y...

已定：
- 主角失去儿子
- 每周三 11 点是定点时刻
- 核心张力 = 说 / 不说

待决：
- 结局走向（开口 / 沉默 / 不告而别）
- Y 的独立 arc 要不要做
```

### B. 给制片人 / 导演的 1 页 pitch

**特点**：要锐，要让对方 1 分钟读完产生兴趣

**包含什么**（按需）：
- Logline（锐的版本）
- Synopsis（300-500 字 prose）
- 长度 / 类型建议（**创作者自己定**，AI 不替他写）

**例子**：
```
# 雨夜便利店

一个失去儿子的老人每周三在便利店遇到陌生人 Y，老人发现 Y 是儿子心脏的受赠者，但选择不告诉他。

[300 字 synopsis]

—— [类型/长度建议由创作者填]
```

### C. 给编剧 / 团队的 development bible

**特点**：要深，给下一棒能直接接着写

**包含什么**（按需）：
- Logline + Premise（详细）
- Synopsis（扩展，800-1500 字）
- 角色卡（主要 + 关键配角）
- 关键场景（如果创作者构思过）
- 风格参考（**创作者主动提的**，不是 AI 推荐的）

### D. 极简一页

**特点**：朋友闲聊问"你最近在搞什么"，5 分钟讲完

**包含什么**：
- Logline 一句
- 主角 / 冲突 / 结局 各一句

## 几种类型的偏好（参考，非强制）

| 类型 | 重点 | 不重要 |
|---|---|---|
| 短剧（中国市场） | Logline 必有钩子；Synopsis 拆成单集 1-3 句 | Premise 学院模板 |
| 电影长片 | 三幕结构隐含；主题陈述 | 分集大纲 |
| 网剧 | 集数大纲 + 单集主线 | 单镜头细节 |
| 文艺片 | Synopsis 偏氛围；主角内心驱动 | 钩子 / 反转 |
| 短视频 | 极简 + 一句钩子 | 详细 Synopsis |

## Premise 模板的取舍参考

如果你要给创作者 Premise（不是每次都要给），**按故事性质取舍**，**不要把所有项都填满**：

| 故事性质 | 推荐字段 |
|---|---|
| 商业短剧 | 钩子 + 反转密度 + 单集结构 |
| 文艺片 | 主题 + 主角内心驱动 + 情感落点 |
| 群像戏 | 每个角色独立动机 + 主线串联方式 |
| 类型片（悬疑/恐怖等） | 类型核心爽点 + 钩子 + 关键转折 |

学院派 8 项（类型/主角/缺失/关键关系/冲突/张力/钩子/赌注）是**最全版本**，不是必填表。

## 写作 tips（让 AI 写的东西不像 AI）

### Logline
- ❌ "一个 [形容词] 的 [名词] 必须 [动词] 否则 [后果]" —— AI 默认模板，太套路
- ✅ 让具体细节带出钩子。"一个老人在便利店等死去的儿子" > "一个失去儿子的老人寻找救赎"

### Synopsis
- ❌ 流水账（"主角做了 A，然后做 B，然后做 C"）
- ✅ 一个**核心动作**贯穿（"X 守着这个秘密整整一年，直到 Y 即将离开"）
- ✅ 至少有 1 个具体细节（一个场景 / 一个画面 / 一句台词）——这是 AI 最容易跳过的部分

### Premise / 角色卡
- ❌ "他必须克服内心的恐惧才能..." —— 抽象，没用
- ✅ "他在便利店买烟时手会抖" —— 具体行为，可演

## AI 给创作者选项时的话术参考

**不要**直接问"你要哪种格式"——大多数创作者不知道格式名字。

**改问用途**：

> "想整理给谁看的？我可以做几种——给自己回顾用的快速 summary、给制片人看的 1 页 pitch、或者更详细的 development bible 给团队。"

或者更轻：

> "你想做个什么样的整理？给别人看还是自己用？"

## AI 主动建议整理的时机（克制）

**不要主动说"要不要总结一下"**——这是 AI 焦虑型行为。

只在创作者自己说出意图（"整理 / 总结 / 汇总 / 我要给 X 看 / 这版基本定了"）时才主动提供选项。
