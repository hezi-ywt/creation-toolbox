# 剧本分镜骨架（阶段 2）

把故事拆成「视频语言脚本」时按这些骨架走。**这不是公式**——按戏剧需要调整时长 / 场次数 / 三幕分配。

骨架要配 `narrative-economy.md` 的主次取舍原则使用。

---

## 草稿格式约定

```markdown
# <故事名> · 剧本分镜（草稿）· ~<总长>

**来源**：故事/<名>.md（或对话直接给）
**总长**：~Nmin · M 个场次（暂定）
**基调**：一句话

**人物**：
- 角色 A · 简短特征
- 角色 B · 简短特征

---

## <场次标题> · ~Ns

[散文式画面描述，让人脑补出画面]

镜头：[自然语言镜头语言，不堆 Seedance 术语]
音效：[关键音]
留白：[结尾状态 / 衔接接口]

> 待商榷：[不确定的地方]
```

**关键约定**：
- 用场次**标题**识别（"发现纸条"），**不用序号** → 增删改不重排
- 时长用 `~Ns` 模糊化 → 强调"暂定可调"
- 散文为主，行尾标注（镜头 / 音效 / 留白）不强制字段
- 三幕用 `## 第一幕 / 二 / 三` 分隔（≥ 60s 适用）
- 待商榷用 `> 待商榷：...` 挂出来
- **解决 AI 无记忆**：每场标"留白 / 结尾状态"，下场起头复述

---

## 按总时长选骨架

### 30s · 广告 / 钩子片

**总场数**：2-3 场

**节奏分配**（参考）：
- 钩子 · 5-10s（建立悬念 / 吸引眼球）
- 主体 · 15-20s（核心信息 / 情感打动）
- 收尾 · 3-5s（CTA / 品牌 / 余韵）

**配套场次模板**（在 `shot-prompt-templates.md`）：
- 钩子 → 戏剧化亮相 / 揭示式
- 主体 → 关键道具引入 / 紧张铺垫 / 产品展示
- 收尾 → 抽离结尾

---

### 60s · 短视频 / 教程 / 概念片

**总场数**：4-6 场

**节奏分配**：
- 开场 · 5-10s（吸引）
- 发展 · 25-35s（铺垫 / 核心内容，2-3 场）
- 转折 · 10-15s（高潮 / 反转）
- 收尾 · 5-10s

**配套**：三段式开场 + 揭示式 + 关键道具引入 / 紧张铺垫 + 抽离结尾

---

### 3min · MV / 概念短片

**总场数**：10-15 场

**节奏分配**（建议与音乐节拍对齐）：
- 前奏 · 15-20s（建立氛围）
- 主歌 1 · 50-60s（4-5 个中场）
- 副歌 · 30-40s（高潮，1-2 个长场 + 节奏对比）
- 主歌 2 · 40-50s
- 桥段 + 收尾 · 30-40s

**配套**：循环使用多模板，与音乐节拍变化对齐

---

### 5min · 微电影 / 故事短片

**总场数**：12-18 场

**节奏分配**（小三幕）：
- **第一幕 · 铺垫** · ~75-90s（4-5 场）
  - 开场建立 · 15-20s
  - 介绍主体 / 触发事件 · 40-60s
  - 决意 / 行动起点 · 10-15s
- **第二幕 · 发展** · ~90-120s（5-7 场）
  - 进入新境 · 20-30s
  - 关键发现 / 阻碍 · 40-60s
  - 转折点 · 20-30s
- **第三幕 · 高潮 + 收尾** · ~90-120s（3-5 场）
  - **情感爆发 · 30-50s**（**全片最长场**，给足凝视时间）
  - 反转 / 危机 · 20-30s
  - 抽离 / 余韵 · 30-40s

**配套**：完整 10 个模板都可能用到，按戏剧功能匹配

---

### 10min · 短剧 / 独立短片

**总场数**：20-30 场

**节奏分配**（完整三幕）：
- 第一幕 · ~2.5min（建立世界 + 角色 + 冲突）
- 第二幕 · ~5min（发展 + 反转 + 危机）
- 第三幕 · ~2.5min（高潮 + 收尾）

**注意**：10min 接近"短片"长度，可能有多线索 / 配角 / 子情节。**建议拆两次**：先 outline 主线、再细化场次。

---

## 拆解执行流程

每次接故事开始拆分镜时，按这步走：

1. **读故事**（已定段为主，或用户对话直接给）
2. **协商总时长 + 类型** → 在上面选骨架
3. **加载 `narrative-economy.md`** 的主次取舍原则
4. **不机械均分** —— 关键场拉长、过场极短、有些场用侧面 / 事后省略
5. **输出草稿态剧本分镜**（散文 + 行尾标注 + 待商榷挂出）
6. **等用户审核** —— 不要主动转 prompt
7. **审核通过后** 才用 `shot-prompt-templates.md` 转 Seedance prompt（阶段 3）

---

## 反面教材

❌ "30s 故事 = 3 场 × 10s 平均" —— 机械均分
❌ 不分幕直接列 15 个平铺场次（5min+ 不分幕 = 创作者审起来累）
❌ 直接出最终格式（标准化的"场次 01/02/03"+ 严谨字段），让用户改起来麻烦

✅ 30s 故事 = 钩子 8s + 主体 18s + 收尾 4s（不均分）
✅ 5min 故事 = 三幕清晰 + 各幕内场次长短交错
✅ 草稿态：散文 + 待商榷 + 时长 `~Ns`，方便迭代
