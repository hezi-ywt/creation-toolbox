# storyboard-drafter

## 核心理念

AI 视频不是堆术语，是**用导演思维拆故事**。

工程约束：
- AI 视频模型一次只能生成 5-15s（Seedance 上限）
- 5 分钟故事 = ~15-20 次独立生成 + 后期剪辑
- 所以创作者要在"故事 → 提示词"之间**有一个中间产物**：让人能在脑子里出画面的剧本分镜（**审稿层**）

本 skill 做**审稿层**——不出 prompt（那是 seedance-prompt 的事）。

## 三个原则

1. **审核 gate 不可跳**：剧本分镜是草稿，必须给用户审核 + 反馈 + 调整。审过才能转 seedance-prompt。
2. **不机械均分时长**：关键场拉长（30-60s）/ 过场极短（3-10s）/ 节奏对比
3. **草稿可改**：场次用标题不用序号 / 散文为主 / 待商榷挂出

## 三个核心行为

### 行为 ①：接故事 → 拆剧本分镜

**触发**：用户给故事 / 想法 / 提到 story-seed 的故事文件 / 描述场景

**做**：
1. 读故事（从 `故事/<名>.md` 已定段，或从 `故事/<名>/` 目录的 index.md + 角色 + 场景大纲）
2. 协商总时长（30s / 60s / 3min / 5min / 10min）
3. 加载 `references/narrative-economy.md` —— 主次取舍 / 省略 / 侧面 / 节奏对比原则
4. 加载 `references/script-to-storyboard.md` —— 按总时长选骨架
5. 输出**草稿态剧本分镜**（散文为主、不用序号、时长不均分）
6. 等用户审核反馈

### 行为 ②：用户审核反馈 → 调整

**触发**：用户对剧本分镜提反馈（保留 / 删除 / 调整 / 合并 / 换顺序 / 改时长）

**做**：
- **直接改**草稿，不阻塞
- 历史不删（保留原版本在文件尾的"调整历史"小节）
- 改完再问："再审一轮还是这版通过了？"

### 行为 ③：通过 → 转 seedance-prompt

**触发**：用户明示"通过 / 这版可以 / 转 prompt / 出 Seedance"

**做**：
1. 把剧本分镜（已审过版本）作为输入
2. **转交 seedance-prompt skill** —— 不要自己出 prompt
3. seedance-prompt 会按场次模板 + 5 维度参考转 Timeline prompt

## 五大原则（拆分镜时必守）

### 1. AI 无记忆

每个场次 prompt 独立生成（这是 seedance-prompt 阶段的事，但**剧本分镜要为此准备**）。

剧本分镜每场标"留白 / 结尾状态"，下一场起头能复述上场结尾状态（如"男主立于停车场中央，深色西装，手中展开的纸条字迹未完全露出"）。

### 2. 时长不平均

机械均分（每场 20s）= 廉价 AI 思维。真实剧本：
- **关键场**（高潮 / 决定瞬间 / 情感爆发）→ 30-60s
- **过场**（开车前往 / 走过去）→ 3-10s
- **节奏对比**：30s 平静 + 5s 爆发才有戏剧感

### 3. 主次取舍

每场问：能不能**省略**？能不能**侧面描写**（反应 / 声音 / 物件 / 画外）？能不能**事后提及**？

详见 `references/narrative-economy.md`。

### 4. 草稿格式约定

```markdown
# <故事名> · 剧本分镜（草稿）· ~<总长>

**来源**：故事/<名>.md
**总长**：~Nmin · M 个场次（暂定）
**基调**：一句话

**人物**：
- 角色 A · 简短特征
- 角色 B · 简短特征

---

## <场次标题> · ~Ns

[散文式画面描述，让人脑补出画面]

镜头：[自然语言镜头语言，不堆 Seedance 术语]
音效：[关键音]
留白：[结尾状态 / 衔接接口]

> 待商榷：[不确定的地方]
```

- 用场次**标题**识别（"发现纸条"），**不用序号** → 增删改不重排
- 时长用 `~Ns` 模糊化 → 强调"暂定可调"
- 散文为主，行尾标注（镜头 / 音效 / 留白）不强制字段
- 三幕用 `## 第一幕 / 二 / 三` 分隔（≥ 60s 适用）
- 待商榷用 `> 待商榷：...` 挂出来

### 5. 不出 Seedance prompt

本 skill 输出**人读的视频语言脚本**（散文 + 自然语言镜头描述）。

不出 `[0s] high angle wide shot...` 这种 Timeline 格式 prompt——那是 seedance-prompt skill 的事。

## 边界

**做**：
- 故事 → 剧本分镜（视频语言脚本草稿）
- 用户审稿对话（保留 / 删除 / 调整 / 合并 / 换顺序）
- 应用叙事经济学（主次取舍 / 省略 / 侧面 / 节奏对比）

**不做**：
- ❌ **出 Seedance Timeline prompt** —— 交给 seedance-prompt skill
- ❌ **写故事 / 改世界观 / 创建角色卡** —— 那是 story-seed / story-assets
- ❌ **跳过审核直接奔 prompt** —— 剧本分镜必须给用户审
- ❌ **机械均分时长** —— 按戏剧节奏分配

## 跟其他 skill 衔接

- **输入来自 story-seed / story-assets**：
  - 单文件简单故事：读 `故事/<名>.md`
  - 项目化故事：读 `故事/<名>/index.md` + `角色/` + `场景大纲/`
- **输出转 seedance-prompt**：用户审过的剧本分镜 → seedance-prompt 出 Timeline prompt
- **由 creation-flow 总调度**

## 衡量做得好不好

1. **审核 gate**：剧本分镜出来后停下来等用户，没奔 prompt？
2. **时长按戏剧节奏分配**：关键场拉长、过场极短，没机械均分？
3. **草稿可改**：用场次标题 / 散文 / 待商榷挂出，便于用户改？
4. **主次取舍真用**：能省的省了、能侧面的侧面了、能事后提的没正面拍？
5. **AI 无记忆衔接**：每场结尾状态标了让下一场能复述？

五个都"是"才算成功。
