# 场景大纲参考

场景大纲是**故事到拍摄之间的桥**——它已经不是"想法"，但还不是"剧本分镜"。把故事拆成**一个个可拍场次**，每场答清楚 "**谁 / 在哪 / 干什么 / 戏剧功能**"。

## 两档详细度（按故事阶段选）

### A. 精简 outline（一场一行）

适合：故事还在迭代，框架先稳。

```markdown
# 主线场景大纲

## 第一幕：触发
1. **发现纸条** · 黄昏 · 停车场 · 男主 · 看到挡风玻璃下的纸条 → 顿悟
2. **驾车前往** · 夜 · 公路 · 男主 · 紧握方向盘
3. **抵达地址** · 夜 · 旧公寓 · 男主 · 观察 → 上楼

## 第二幕：进入
4. **开门对视** · 夜 · 公寓门口 · 男主 + 陌生女子 · 沉默对望
...
```

格式：`编号 · 场名 · 时间 · 地点 · 出场人 · 一句话概括`

**优点**：快速调顺序、删场、合并；脑里能秒过一遍故事
**缺点**：信息少，不够拍

### B. 详细场次卡（一场一节）

适合：outline 稳了，要交给 ai-video-prompt 拆分镜。

```markdown
## 发现纸条

**戏剧功能**：触发事件 · 把男主推出舒适区
**时长建议**：~15s
**时间地点**：黄昏 · 城市边缘空旷停车场
**出场角色**：男主
**出场道具**：白色折叠纸条 · 旧黑色轿车
**冲突 / 转折**：男主从"日常下班"瞬间变"被未知信息击中"
**情绪曲线**：疲惫 → 注意到异常 → 犹豫 → 顿悟震惊
**结尾状态**：男主立于车前，手中展开纸条字迹未完全露出，瞳孔放大
**前接 / 后接**：前接「办公室散场」/ 后接「揭示纸条」
```

**关键字段**：戏剧功能 / 时长 / 角色 / 转折 / 情绪曲线 / 结尾状态

## frontmatter

参见 [asset-rules.md](./asset-rules.md)：

```yaml
---
title: 主线场景大纲
type: 剧本                # 单片 / 单集 = 剧本
status: 草稿
episode: E01              # 剧本类型必填
tags: [第一季, 主线]
created: 2026-05-17
modified: 2026-05-17
---
```

多集的话每集一个文件：`场景大纲/E01.md` `E02.md` ...

## 跟 ai-video-prompt 的衔接 ⭐

场景大纲是 **ai-video-prompt 阶段 2「剧本分镜」的直接输入**。两者关系：

```
story-creator                    ai-video-prompt
─────────────────                ─────────────────
场景大纲（一场 = 一个戏剧单位）  →  剧本分镜（一场拆成 timeline beats）
"发现纸条 · 15s · 男主在停车场..."     "[0s] 高角度远景 [4s] 推近 [8s] 特写..."
```

**移交标准**：场景大纲达到"详细场次卡"级别（含戏剧功能 / 时长 / 角色 / 结尾状态），就可以交给 ai-video-prompt 拆分镜了。

**两个 skill 的分工边界**：
| story-creator 负责 | ai-video-prompt 负责 |
|---|---|
| 这一场要不要拍（戏剧必要性）| 这一场怎么拍（镜头 / 光线 / 运镜）|
| 一共几场（拆解粒度）| 每场拆成几个 beat（15s 内） |
| 场次之间的故事衔接 | 场次之间的视觉衔接（AI 无记忆） |

## 怎么从故事拆到场景大纲

### 步骤
1. 读 `index.md`（故事核心）
2. 列**戏剧节拍**（开始 / 触发 / 发展 / 高潮 / 结尾）——参考 [beat-structures.md](./beat-structures.md)
3. 每个节拍拆成 1-N 个**可拍场次**
4. 检查 [narrative-economy.md](../../../skills/ai-video-prompt/references/narrative-economy.md) 的取舍原则（能不能省略 / 侧面 / 事后提及）
5. 输出精简 outline（一场一行）让用户审
6. 审过的场次升级为详细场次卡

### 时长配额（不机械均分 · 关键场要拉长）

参考 ai-video-prompt 的 `narrative-economy` —— 5 分钟故事典型分布：

| 场次类型 | 时长 | 数量占比 |
|---|---|---|
| 极短过场（开车前往 / 走过去）| 3-8s | ~30% |
| 中场（建立 / 铺垫）| 10-20s | ~40% |
| 长场（关键 / 转折）| 25-40s | ~20% |
| 极长场（情感爆发 / 高潮）| 40-60s | ~10% |

详见 [narrative-economy.md](../../../skills/ai-video-prompt/references/narrative-economy.md)。

## 反面教材

❌ **每场时长一样**（每场 20s × 15 场）—— 机械均分，戏剧密度低
❌ **场次都是「主角去 X 做 Y」**（缺戏剧功能描述）—— 拍出来观众不知道为什么
❌ **场次跨度太大**（一场 60s 但故事跳了 2 天）—— 拆成多场
❌ **场次过细**（"男主走出门"一场，"男主下楼"另一场）—— 合并

## 用法

### 在 story-creator 工作流里
- **行为 ④ 升级项目化**触发时，创建 `场景大纲/主线.md`
- 用户讨论"这场要不要"时，AI 帮维护这个文件
- 改场次顺序 / 增删时，注意更新场次编号 + 后续场的 `前接 / 后接`

### 关键提示
- **outline 是协作工具**——不是导演分镜稿。该模糊就模糊（剧情未定的场用"待商榷"标）
- **场次有戏剧功能才存在**——"日常生活"型场次必须能服务情感 / 反差 / 角色发展
- **过场极短 / 关键场极长**——不要平均
