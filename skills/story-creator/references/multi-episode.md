# 多集架构参考

故事**单片 ≤ 10 分钟用单文件**。**超过 10 分钟、或本就是分集**（短剧 / 微短剧 / 季播）时升级为多集架构。

## 何时拆集

| 故事类型 | 用单片 | 用多集 |
|---|---|---|
| 单片 ≤ 5 min | ✅ 单文件 | ❌ |
| 微短剧 1-2 min/集 × 30+ 集 | ❌ | ✅（每集一个文件）|
| 季播 20-50 min/集 × 12-24 集 | ❌ | ✅（每集一个文件 + 季总览）|
| 长片 90+ min | ⚠️ 可单文件但分幕 | ✅（每幕一个文件）|

## 文件组织

```
故事/<故事名>/
├── index.md                  # 故事总览（含分集表）
├── 季线/                      # 总线设计（季 / 多季 / 完结）
│   └── S1-总览.md
├── 集线/                      # 每集一个文件
│   ├── S1E01.md
│   ├── S1E02.md
│   └── ...
├── 角色/                      # 跨集共享（演变随时间）
├── 世界观/
└── 草稿/
```

**关键决策**：
- 跨集的**角色**放 `角色/` 根目录（不分集）—— 全部集都看同一份
- 跨集的**世界观**放 `世界观/`（不分集）
- 单集的**场次列表**放 `集线/S1E01.md` 里（每集独立）

## 集线 vs 季线

### 集线（每集 1 个文件）

```markdown
---
title: S1E01 触发
type: 剧本
status: 草稿
episode: S1E01
tags: [第一季, 第一集, 触发集]
created: 2026-05-17
modified: 2026-05-17
---

# S1E01 触发

## 集级戏剧功能
触发主线 + 建立男主形象 + 留钩子

## 时长
~3 分钟

## 角色出场
- 男主（主角，全集）
- 老王（配角，开头 30s）

## 场景大纲
（按 scene-outline.md 的格式列）
1. 开场 · 办公室 · ...
2. 离开 · 楼下 · ...
3. 发现纸条 · 停车场 · ...
...

## 集间钩子
- **前接**：（首集，无）
- **后接**：S1E02 揭示纸条内容 → 男主开车前往
- **本集留白**：纸条内容未揭示，男主决心未明示
```

### 季线（1-N 个文件，看复杂度）

```markdown
---
title: S1 总览
type: 故事小说
status: 草稿
tags: [第一季, 主线]
created: 2026-05-17
modified: 2026-05-17
---

# 第一季总览

## 主线 logline
中年男主收到陌生纸条，揭开妻子"死亡"骗局，逐步发现自己生活的虚假性

## 季级弧光
**男主**：克制疲惫 → 决意调查 → 接受真相 → 重新连接女儿

## 集数 + 高潮点
共 8 集：
- S1E01-02 触发 + 揭示纸条
- S1E03-04 第一调查段（找到地址）
- **S1E05 中点反转**（妻子在但拒见）
- S1E06-07 高潮调查段（揭示真正原因）
- **S1E08 季终**（接受 + 钩 S2）

## 副线
- 女儿小满的副线：从冷漠 → 信任父亲（贯穿全季）
- 老王的副线：起初只是朋友，S1E06 揭露他知道部分真相
```

## 多集创作的 4 个核心问题

### 1. 集间钩子（cliffhanger）

每集结尾要给观众"必须看下一集"的理由：

| 钩子类型 | 例 |
|---|---|
| **未解之谜** | 揭示一半信息，剩一半下集 |
| **关键决定瞬间** | 角色刚要做决定，cut |
| **新角色 / 反转登场** | 镜头转到一个谁也没见过的人 |
| **危险临近** | 角色不知道但观众知道有事要发生 |

**避免**：每集都是"惊讶脸 + 切" —— 观众会麻木。混用不同类型。

### 2. 主副线交织

| 集类型 | 主线 | 副线 |
|---|---|---|
| 触发集（S1E01）| ~80% | ~20%（暗示副线存在）|
| 发展集（S1E02-06）| ~60-70% | ~30-40% |
| 高潮集（S1E07-08）| ~80% | ~20%（副线收 / 借力主线）|

副线**不是补丁**——它要在某集真正影响主线（如 S1E04 女儿副线推动男主做某个决定）。

### 3. 角色弧光跨集分布

弧光按"集"拆解：

```markdown
## 男主弧光（跨 S1 8 集）

- S1E01-02：克制 + 触发（看到纸条决意调查）
- S1E03-04：行动期（开始查 + 试探女儿）
- S1E05（中点）：信念崩塌（妻子还活但拒见）
- S1E06-07：转变期（找到真正原因 + 学会接受）
- S1E08：新平衡（重新连接女儿）
```

**每集要明确**：本集角色比上集 advance 了什么？没 advance 的集是水集。

### 4. 留白节奏

**别每集都揭示新信息**——观众消化不过来。

参考节奏：

```
S1E01: 提出问题
S1E02: 揭示一层
S1E03: 制造新疑问
S1E04: 揭示一层 + 制造新疑问
S1E05: 中点反转（颠覆已知）
S1E06: 揭示中点反转的解释
S1E07: 揭示最大谜底
S1E08: 处理后果 + 钩 S2
```

## 跟 ai-video-prompt 衔接

每集是 ai-video-prompt 的"一份完整故事输入"。AI 视频拆分镜时一次处理一集：

```
集线/S1E01.md
   ↓ 喂给 ai-video-prompt
[剧本分镜] S1E01 拆成 8-15 个场次
   ↓ 用户审核
[Seedance prompt] 每场次输出
   ↓ 用户拿 prompt 在 Seedance 生成 → 剪辑成 3 分钟视频
```

## 反面教材

❌ 每集独立无钩子 → 观众看完一集没动力看下一集
❌ 主线一直推动副线没存在感 → 角色单薄
❌ 中点反转放结尾 → 后面没戏可演
❌ 角色弧光集中在结尾两集 → 中间所有集都白看
❌ 每集都揭示重大信息 → 观众消化不过来 / 后期没料

## 用法

### 在 story-creator 工作流里
- **行为 ④ 升级项目化**触发时，**先问用户**：单片 / 短剧 / 季播？
- 选短剧 / 季播才建 `季线/` `集线/` 目录
- 单片不需要分集架构，沿用单文件 + `场景大纲/`

### 集间衔接 hygiene
- 改 S1E03 的设定时，AI 主动 grep S1E04+ 看是否需要同步（参 [asset-hygiene.md](./asset-hygiene.md) 规则 3）
- 每集 frontmatter 的 `episode` 字段必须唯一
