# 角色卡参考

角色是故事的最小情感单元——故事不是事件的串联，是**角色面对事件的反应**串联。一个好角色卡不是设定堆砌，是**让 AI 在写场景时能快速调出"他会怎么反应"**。

## 4 个层次（按写作优先级）

按这个顺序写——前一层定了才写下一层，否则容易堆砌无效设定。

### 1. 基础（外在 · 给眼睛看的）
- 名字 / 年龄 / 性别
- 外貌（**让人脑里能出画面的细节**，不要"高大英俊"这种空话）
- 职业 / 阶层
- 标志性物件 / 习惯（如戴金属表 / 紧张时摸表盘）

### 2. 心理（内在 · 给戏剧用的）
- **核心欲望**（他真正想要什么——不是表面目标）
- **核心恐惧**（他不敢面对什么）
- **价值观**（什么对他不可妥协）
- **盲点**（他自己看不见的弱点 / 偏见）

### 3. 关系（结构性 · 给冲突用的）
- 关键关系人物（父母 / 爱人 / 死敌 / 引路人）+ 关系性质
- 跟其他角色的张力（爱 / 恨 / 依赖 / 嫉妒）
- 关系如何随故事变化

### 4. 弧光（character arc · 给故事用的）
- 故事**开始**时他是什么样
- 故事**结束**时变成什么样
- 转变的**触发**事件
- 转变的**阻力**（他不愿意变的原因）

```markdown
# 林默

## 基础

- **年龄**：45
- **外貌**：身高 178，发际线后退到额前 3 厘米，左手腕戴一块磨损的银色手表（前妻送的）
- **职业**：广告创意总监（4A 公司，第 19 年）
- **标志习惯**：紧张时摸手表表盘
- **服装**：深灰西装外套常年不换，里面 T 恤换季节

## 心理

- **核心欲望**：被妻子看见——不是被夸赞，是被理解他这 5 年怎么过来的
- **核心恐惧**：发现妻子的"死"是骗局的话，自己 5 年的悲痛就成了自我感动
- **价值观**：不主动给人添麻烦（他妈传的）
- **盲点**：把"克制"当成"成熟"，错过了 5 年来女儿其实需要他更外放的爱

## 关系

- **妻子（林夏）**：5 年前"病逝"。其实没死，被某种势力保护起来。男主完全不知情
- **女儿（林小满，12 岁）**：跟男主同住，但关系疏远——她记得妈妈，男主从不主动提
- **死党（老王）**：广告圈大学同学，知道男主所有事

## 弧光

- **开始**：克制、疲惫、把"妻子已死"作为已定事实，每天上班 → 接女儿 → 沉默
- **触发**：收到挡风玻璃下的纸条「你妻子并没有死」
- **阻力**：他害怕如果是真的，那 5 年的克制就是个笑话——克制反而成了不敢面对真相的借口
- **结束**：找到妻子但接受她无法回到日常生活；学会跟女儿外放地谈起她妈

---

## 待商榷

- 林夏为什么不死？被什么势力？这个设定决定故事类型（悬疑 / 政治 / 超自然）
- 老王这个配角要不要做？还是不必要直接砍
```

## 反面教材

❌ **堆设定但跟故事无关**：星座 / 血型 / 喜欢的颜色——除非故事真用到
❌ **完美角色**：所有角色都该有"挡住他成长的东西"，零缺点 = 没戏
❌ **静态角色**：弧光那段空着的角色，剧情就不会动他
❌ **跟主线无关的过去**：童年阴影只在跟主线相关时写
❌ **抽象描述**："性格内向"不如"被陌生人搭话时假装在看手机"

## 用法

### 在 story-creator 工作流里
- 当创作者从单文件故事**升级**到多角色项目时（行为 ④），按这个模板创建 `角色/<名字>.md`
- 每次跟 AI 讨论某个角色时，AI 先 Read 这个文件加载到上下文
- 角色卡 modified 时间反映"最近被讨论"

### 跟 ai-video-prompt 衔接
- ai-video-prompt 拆剧本分镜时会 Read 角色卡，把 `外貌` / `标志习惯` 字段抽出来填 prompt 的"资产"段（解决 AI 无记忆问题）
- 因此外貌字段**必须可视化**（能直接转 Seedance prompt 的描述）

### 改角色名 / 设定时
走 [asset-hygiene.md](./asset-hygiene.md) 的改名联动流程：grep 找所有引用 → 列给用户 → 逐条决定。
