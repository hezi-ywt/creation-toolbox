# Quickstart · 5 分钟跑通完整流程

跟着这个例子用 6 个 skill 把一个故事种子做成可粘贴的 Seedance AI 视频 prompt。

## 故事种子（用这个跑）

> 24 小时便利店深夜，独居白领发现总有一只**黑猫**准时来同一个时间趴在自动门外。试了几次想喂它，它都不进店，只是看着她。偶然发现这只猫脖子上挂着一块她**男朋友 3 年前失踪当晚戴的手表**。但她男朋友「失踪」前一周明明已经分手了。

## 整个流程

5 个阶段，每阶段对应一个 skill。第一个 `creation-flow` 是总调度——你不用关心切换，它自动判断：

```
你给种子 → creation-flow 切到 story-seed → 讨论出 logline / synopsis
你加角色 → creation-flow 切到 story-assets → 项目化 + 角色卡
你说做视频 → creation-flow 切到 storyboard-drafter → 拆剧本分镜（草稿）
你审核通过 → creation-flow 切到 seedance-prompt → Timeline prompt
```

整个过程你**只跟一个合伙人聊**，背后是 6 个 skill。

---

## 阶段 1：故事种子讨论（`story-seed`）

### 你说

> "我有个故事想法：24 小时便利店深夜，独居白领发现总有一只黑猫准时来同一个时间趴在自动门外..."

### AI 自动做

- 识别这是「粗浅大致故事线」类种子（已有开头-关键事件-走向）
- **不走完整挖矿**（不该问 50 个问题）
- 结构化复述确认理解
- 只挖未尽部分（3-4 个关键决定，如"失踪是真失踪还是引号失踪"）

### 产出

文件 `故事/<故事名>.md`：

```markdown
# 猫不进来

## 已定
（暂无）

## 草稿
### 起点（2026-05-18）
[你的种子原样记录]
```

---

## 阶段 2：复杂化触发项目化（`story-assets`）

### 你说

> "我想加女主（妻子）+ 闺蜜，做成 3 集短剧"

### AI 自动做

- 识别项目化触发条件（多角色 + 多线索 + 分集）
- **不自作主张升级** —— 先跟你确认
- 同意后把 `故事/<名>.md` 升级为 `故事/<名>/` 目录
- 创建角色卡，AI 自动补的细节**标占位符**："独居白领（占位名，可改）"
- **探测 `asset-index` CLI**：装了告诉你"会跑 check 验证"，没装明示跳过

### 产出

目录结构：

```
故事/猫不进来/
├── index.md                # 故事核心 + 已定段
├── 角色/
│   ├── 独居白领.md         # 含 frontmatter（type/status/tags）
│   ├── 妻子.md
│   └── 闺蜜.md
└── 场景大纲/（按需）
```

角色卡示例：

```yaml
---
title: 独居白领（占位名，可改）
type: 角色
status: 草稿
tags: [女主, 第三者]
created: 2026-05-18
modified: 2026-05-18
---

# 独居白领

## 基础
- 年龄：（占位 · 28? 32? 35?）
- 职业：（占位 · 影响她下班的时间）
...
```

## 阶段 3：进入视频阶段（`storyboard-drafter`）

### 你说

> "OK 故事够用了，把它做成视频吧 / 拆分镜"

### AI 自动做

- 识别进入视频阶段信号
- 协商总时长（30s / 60s / 3min / 5min / 10min）
- 应用**「叙事经济学」**：不机械均分时长 —— 关键场 30-60s，过场 3-10s
- 输出**剧本分镜草稿**（视频语言脚本，让人脑补画面）
- 等你审稿反馈

### 产出（示例）

```markdown
# 猫不进来 · 第 1 集 · 剧本分镜（草稿）· ~60s

## 关掉的便利店灯 · ~8s

凌晨 1 点的 24 小时便利店，玻璃门内一片冷白光。她蹲在收银台后看手机，
外头空无一人。一只黑猫在自动门外停下，准准坐成一个安静的弧度，看进店里。

镜头：定镜远景透过玻璃门拍内+外
音效：荧光灯电流嗡鸣 / 远处车流
留白：猫坐着没动，她回到手机

> 待商榷：这是第一次出现，还是已经是"惯例"了的某一晚？

## 看见手表的那一秒 · ~32s
...（关键场拉到 32s，过场 8s，节奏对比）
```

**关键**：阶段 3 **不出** Seedance Timeline prompt（那是阶段 5 的事）。这一步是给**你**审稿用的视频语言脚本。

---

## 阶段 4：用户审稿

### 你说

> "通过" / "第 2 场太慢" / "把闺蜜删了" / 等等

### AI 自动做

- 不阻塞地直接改
- 历史保留（草稿尾"调整历史"小节）
- 改完再问"再审一轮还是这版通过？"

**审过的版本才能转阶段 5。** 这是设计上的硬 gate —— 不让 AI 跳过审核直接奔 prompt。

---

## 阶段 5：转 Seedance Prompt（`seedance-prompt`）

### 你说

> "通过，出 Seedance 提示词"

### AI 自动做

- 严格遵循 **Seedance 6 步公式**（主体 / 动作 / 环境 / 运镜 / 光线 / 风格）
- **8 个官方运镜优先**（push / pull / pan / track / orbit / aerial / handheld / fixed）
- **AI 无记忆约束**：每段 prompt 起头**重新描述**人物 + 资产
- **32s 场次自动拆 3 段**（Seedance 5-15s 上限）
- 建议 **`@image1` 参考图锁脸**避免角色漂移
- **avoid 来源标注**：用户的已定决策（如"不让看清表盘"）转成 avoid 时显式说明出处

### 产出（一个场次示例）

```
[0s] 凌晨 24 小时便利店内景，30 岁上下东亚女性，黑发齐肩，
     穿米白色长袖，坐在收银台后低头看手机，冷白色荧光灯。
     一只黑色短毛猫在自动门外端坐，正对镜头。
     镜头：固定远景（fixed wide），透过玻璃门构图，
     门框居中将画面分成内外两层。
     光线：店内冷白色荧光（5500K），外部夜色微蓝。
     浅景深，焦点在猫。
     时长 8 秒。风格：cinematic 35mm film grain, 都市疏离感。
     避免：fast motion, dramatic music swell。
```

复制粘贴到 Seedance / 可灵 / Veo / Sora 平台即可生成视频。

## creation-flow 在背后做了什么（你不用看，但好奇时看）

虽然你感觉是"一个合伙人"，`creation-flow` 在编排：

| 编排动作 | 例子 |
|---|---|
| **先固化再切换** | 阶段 1 的"已定"在切阶段 2 前先写入 index.md，不丢 |
| **披露弹药状态** | 阶段 3 拆分镜前主动告诉你"妻子卡未填会导致妻子线偏淡"，不阻塞但让你知情 |
| **跨阶段已定回写** | 阶段 5 出 prompt 时识别"这条决定约束 2、3 集"，自动回写阶段 1 的 index.md |
| **跨会话 TODO 持久化** | 留待办时自动写入 index.md "待办"段，下次回来 AI 先汇报 |

## 关键设计原则（贯穿 6 个 skill）

- **创作者主权**：你做决定，AI 给选项
- **AI 无记忆**：跨 prompt 必重述人物 / 资产
- **不机械均分**：关键场 30-60s，过场 3-10s
- **草稿可改**：场次用标题不用序号，散文为主，待商榷挂出
- **资产即文件**：角色 / 世界观 = .md + frontmatter，git 友好
